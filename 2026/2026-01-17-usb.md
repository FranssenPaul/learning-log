# USB: Essential Guide

Quick reference for understanding USB hardware and software.

---

## What is USB?

**USB (Universal Serial Bus)** is a communication protocol for connecting devices to computers.

- Physical side: cables, connectors, electrical signals
- Logical side: data format, enumeration, communication rules
- One USB controller can expose multiple buses

```bash
lspci | grep -i usb
```

---

## How it works

### Connection process
1. Device is plugged in and receives power.
2. The device signals its speed on D+ or D-.
3. The host sends a reset.
4. The host asks for descriptors.
5. The host assigns a device number.
6. The kernel loads the right driver.

### Communication model
- USB is host-driven.
- The host polls devices.
- Devices answer with data or with "nothing to send".

## Device identification

### Reading `lsusb`
```text
Bus 001 Device 005: ID 046d:c336 Logitech, Inc.
│       │           │    └─ Product ID
│       │           └─ Vendor ID
│       └─ Device number
└─ Bus number
```

- Vendor ID identifies the manufacturer
- Product ID identifies the device
- Device class hints at the type of driver needed

## Speeds and bandwidth

| USB Version | Speed | Real-World |
|---|---|---|
| USB 2.0 | 480 Mbps | ~40 MB/s |
| USB 3.0 | 5 Gbps | ~400 MB/s |
| USB 3.1 | 10 Gbps | ~900 MB/s |
| USB 3.2 | 20 Gbps | ~1.8 GB/s |

Devices on the same bus share total bandwidth.

## Drivers and commands

Built-in drivers often cover HID, storage, video, and audio devices.

```bash
lsusb
lsusb -t
lsusb -v
sudo dmesg -w
lsblk
lsmod | grep usb
```

---

## Quick troubleshooting

- Not detected: test another port or cable, then check `lsusb`
- Detected but broken: inspect `dmesg` and loaded modules
- Storage not mounted: verify with `lsblk`, then mount manually if needed

---

## Vocabulary

**Vendor** means the company that sells or manufactures the device.
