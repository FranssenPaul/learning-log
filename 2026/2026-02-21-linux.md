# Linux — Synthèse

> Concepts appris et compris, avec exemples concrets

---

## Kernel and syscalls

The Linux kernel is the core of the operating system.

- it manages processes, memory, files, hardware, and networking
- user programs reach it through **syscalls**

Typical syscalls:

```text
open()
read()
write()
fork()
exec()
socket()
exit()
```

---

## systemd and processes

- `systemd` is PID 1
- it starts and manages services
- every running program is a process with its own PID
- shells typically use `fork()` and then `exec()`

Useful commands:

```bash
ps aux
pstree -p
systemctl status docker
```

---

## Everything is a file

Linux exposes many things as files:

- regular files
- directories
- devices
- pipes
- sockets
- virtual files in `/proc`

This also leads to the idea of file descriptors:

```text
0 -> stdin
1 -> stdout
2 -> stderr
```

---

## Namespaces

Namespaces isolate what a process can see.

Examples:

- PID namespace
- network namespace
- mount namespace
- user namespace

They are a core building block behind containers.
