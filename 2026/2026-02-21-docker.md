# Docker — Synthèse

## Core concepts

- an **image** is a layered filesystem template
- a **container** is a running instance of that image
- containers share the host kernel, unlike virtual machines

---

## Dockerfile

The Dockerfile is the recipe used to build an image.

```dockerfile
FROM python:3.12-slim
WORKDIR /app
COPY . .
CMD ["python", "main.py"]
```

Useful instructions:

- `FROM`
- `COPY`
- `RUN`
- `WORKDIR`
- `CMD`
- `ENV`

---

## Core commands

```bash
docker build -t mon-app .
docker run mon-app
docker run -p 8000:8000 mon-app
docker ps
docker images
docker logs mon-conteneur
```

---

## Workflow and POS context

Typical flow:

- build locally
- push to a registry or export to USB
- pull or load on the target server
- start with `docker compose up`

For the POS use case, Docker helps package the API and database with a reproducible setup.
