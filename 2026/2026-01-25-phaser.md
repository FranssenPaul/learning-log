# Learning Log — Phaser for Educational Games

## Goal

Use **Phaser** for interactive animations and small games, optionally inside a React app.

---

## Why Phaser

### JSXGraph vs Phaser
- JSXGraph is great for formal math visuals.
- Phaser is better for motion, collisions, input, and game loops.
- For teaching games or animated simulations, Phaser is the better fit.

---

## Physics choices

### Arcade Physics
- fast and simple
- ideal for Pong or arcade games

### Matter Physics
- real rigid-body physics
- ideal for peg collisions and physics demos

---

## Galton board

Two valid approaches:

- physical simulation with Matter Physics
- Bernoulli/random-walk simulation for pedagogy

The second approach is often clearer for students and scales better.

---

## Phaser + React

Rule:

> React owns the UI. Phaser owns the canvas.

Recommended pattern:

- React mounts the game once
- Phaser manages rendering and game logic
- UI controls stay in React
- shared communication goes through refs or events

---

## Routing and publishing

One React app can host many simulations:

- `/galton-bernoulli`
- `/galton-physics`
- `/pong`

Cloudflare Pages works well for this kind of static React + Phaser deployment.
For React Router, a `_redirects` file is needed:

```text
/* /index.html 200
```

## Student-facing catalog

Students should navigate a clear home page, not raw files.

- one card per animation
- one route per experience
- direct links from QR codes or an LMS
