# Vocabulaire des Cards UI

## Important d'abord

Il n'existe pas un vocabulaire "officiel" unique et universel pour les cards UI.

En revanche, il existe un vocabulaire tres courant, partage dans :

- le design d'interface
- le product design
- le frontend
- les design systems

Ce document rassemble ce vocabulaire standard, avec une explication simple et des exemples.

## Qu'est-ce qu'une card

Une `card` est un bloc d'interface autonome qui regroupe un contenu ou une action.

On l'utilise souvent pour :

- presenter une information
- isoler une section
- structurer une page
- mettre en avant une action

Exemples :

- une card de profil
- une card produit
- une card de message
- une card de connexion
- une card de statistiques

## Structure generale d'une card

Une card est souvent decomposee en plusieurs zones.

Toutes ne sont pas obligatoires.

Le schema le plus courant ressemble a ceci :

1. `header`
2. `body` ou `content`
3. `footer`

## 1. Card

Le mot `card` designe le conteneur complet.

Exemple :

- "Cette card contient le titre, le texte et le bouton."

Dans Ducit :

- la card `Chapitre actif`
- la card `Mot de passe`
- la card `Conversation`

## 2. Header

Le `header` est la zone du haut de la card.

C'est souvent la partie qui donne le contexte.

Il peut contenir :

- un petit label
- un titre
- une icone
- un badge
- une action

Exemple :

- en haut d'une card, on affiche `Chapitre actif`
- c'est un element du header

## 3. Title

Le `title` est le titre principal.

C'est le texte le plus important dans la card.

Il sert a dire clairement ce qu'est la section.

Exemples :

- `Mot de passe`
- `Atelier de resolution`
- `Profil eleve`

## 4. Eyebrow

Le mot `eyebrow` est tres utilise en design UI.

Il designe le petit texte place au-dessus du titre principal.

Il sert a :

- introduire le contexte
- categoriser la section
- donner un niveau de lecture plus fin

Visuellement, il est souvent :

- plus petit
- en capitales
- avec un tracking plus large
- moins dominant que le titre

Exemples :

- `Atelier IA`
- `Acces`
- `Chapitre actif`
- `Conversation`

Dans ton interface, `Chapitre actif` ressemble a un `eyebrow`.

## 5. Label

Le mot `label` a plusieurs sens selon le contexte.

Dans un formulaire, un `label` est le texte associe a un champ.

Exemple :

- `Mot de passe` au-dessus d'un input

Dans une card, on utilise parfois aussi `label` pour parler d'un petit texte de categorie.

Mais pour une card, quand le texte est au-dessus du titre, `eyebrow` est souvent le mot le plus precis.

## 6. Subtitle

Le `subtitle` est un sous-titre.

Il accompagne le titre principal.

Il est moins central que le titre, mais plus fort qu'une description longue.

Exemple :

- Titre : `Conversation`
- Sous-titre : `Fonction carre`

Tout le monde n'utilise pas ce mot strictement, mais il est frequent.

## 7. Description

La `description` est le texte explicatif.

Elle sert a aider l'utilisateur a comprendre :

- ce que fait la card
- ce qu'il doit faire
- ce qu'il va se passer

Exemples :

- "Le mot de passe sert seulement a ouvrir une session securisee."
- "Ce menu prepare la suite."

La description est souvent plus longue qu'un sous-titre.

## 8. Body

Le `body` est le contenu principal de la card.

C'est la zone utile, celle qui porte l'information ou l'interaction.

Exemples :

- un champ de saisie
- une liste
- un message
- un graphique
- un select

Dans une card de connexion, le body peut contenir :

- l'input mot de passe
- le texte d'etat

## 9. Content

`Content` est un mot tres proche de `body`.

Beaucoup d'equipes utilisent l'un ou l'autre.

En pratique :

- `body` insiste sur la structure de la card
- `content` insiste sur ce qu'elle contient

Dans un design system, on verra parfois :

- `CardHeader`
- `CardContent`
- `CardFooter`

## 10. Footer

Le `footer` est la partie basse de la card.

On y place souvent :

- des boutons
- des liens
- des metadonnees
- des actions secondaires

Exemples :

- `Annuler`
- `Valider`
- `Voir plus`

Toutes les cards n'ont pas de footer.

## 11. Actions

Les `actions` sont les elements cliquables qui permettent d'agir.

Exemples :

- bouton `Nouvelle conversation`
- bouton `Se connecter`
- lien `Voir le detail`

Une card peut avoir :

- une action principale
- une action secondaire

## 12. Primary Action

La `primary action` est l'action principale de la card.

C'est celle que l'utilisateur est le plus cense utiliser.

Elle est souvent plus visible que les autres.

Exemples :

- `Se connecter`
- `Envoyer`
- `Nouvelle conversation`

## 13. Secondary Action

La `secondary action` est une action utile, mais moins prioritaire.

Elle est souvent plus discrete.

Exemples :

- `Annuler`
- `Retour`
- `Fermer`

## 14. Badge

Un `badge` est un petit element visuel qui signale un etat ou une categorie.

Exemples :

- `Nouveau`
- `Connecte`
- `Brouillon`
- `En attente`

Un badge est court et visuel.

Il ne remplace pas un titre.

## 15. Meta / Metadata

La `meta` ou `metadata` designe les informations secondaires associees a la card.

Exemples :

- une date
- un statut
- un nombre de messages
- un auteur

Ce sont des informations utiles, mais pas centrales.

## 16. Media

Le `media` est un contenu visuel integre a la card.

Exemples :

- une image
- une icone
- une miniature
- une illustration

## 17. Container

Le `container` est le bloc global qui entoure d'autres elements.

Le mot est plus general que `card`.

On peut dire :

- "La card est un container"

Mais tous les containers ne sont pas des cards.

## 18. Variant

Le mot `variant` designe une variation d'un meme composant.

Exemples :

- card simple
- card accentuee
- card interactive
- card selectionnee

Dans un design system, on dit souvent :

- `default`
- `outlined`
- `elevated`
- `compact`

## Petit exemple de lecture d'une card

Imaginons cette structure :

- Eyebrow : `Acces`
- Title : `Mot de passe`
- Description : `Le mot de passe sert seulement a ouvrir une session securisee.`
- Body : champ mot de passe
- Primary action : bouton `Se connecter`
- Status text : texte d'etat sous le bouton

On pourrait donc decrire cette card ainsi :

"Cette card possede un header avec un eyebrow et un title, une description, puis un body de formulaire avec une action principale."

## Les termes les plus utiles a retenir

Si tu veux aller a l'essentiel, retiens surtout ceux-ci :

- `card` : le bloc complet
- `header` : la zone haute
- `title` : le titre principal
- `eyebrow` : le petit texte au-dessus du titre
- `description` : le texte explicatif
- `body` ou `content` : le contenu principal
- `footer` : la zone du bas
- `actions` : les boutons ou liens
- `badge` : le petit indicateur d'etat

## Conseils pratiques pour bien nommer dans un projet

Pour rester coherent, choisis un vocabulaire et garde-le partout.

Exemple de convention simple :

- `card`
- `card-header`
- `card-title`
- `card-eyebrow`
- `card-description`
- `card-content`
- `card-actions`
- `card-footer`

Le plus important n'est pas d'avoir "le mot parfait", mais d'avoir :

- des mots compris par toute l'equipe
- des mots utilises de facon stable
- des mots assez precis pour eviter l'ambiguite

## Resume final

Le vocabulaire le plus standard pour une card UI repose surtout sur :

- `card`
- `header`
- `title`
- `eyebrow`
- `description`
- `content` ou `body`
- `footer`
- `actions`
- `badge`

Si tu veux parler proprement de ta UI actuelle, tu peux deja dire :

- `Chapitre actif` est un `eyebrow`
- `Mot de passe` est un `title`
- `Nouvelle conversation` est une `action`
- la zone des messages est un `content area`

Ce vocabulaire est deja tres propre pour apprendre et pour construire une interface de facon professionnelle.
