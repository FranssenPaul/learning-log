# POS LAN Setup with USB Devices

This note isolates the restaurant POS part from the USB theory.

---

## Example setup

For a compact POS machine with many peripherals:

- USB 2.0 bus: tablets, keyboard, Bluetooth dongle
- USB 3.0 bus: backup drive

The practical idea is simple:

- low-bandwidth devices can share one bus
- storage or backup devices are better on a faster bus

---

## Why it matters

In a restaurant setup, you want:

- stable peripherals
- predictable bandwidth usage
- easy diagnostics before deployment

Even with several tablets connected, the bandwidth usage can stay modest if the devices are well distributed.

---

## Pre-deployment checklist

```bash
sudo dmesg -w
lspci | grep -i usb
lsusb -t
```

- test each port
- verify which controller and bus are used
- label ports once the setup is validated
